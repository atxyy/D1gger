GET /api/agent

Wire your agent against this server’s origin. Read the manifest first — it describes recommended_flow and enumerates artifact sections (subdomains_all, …, summary). Humans use the dashboard; agents consume JSON / plaintext below.

Intent HTTP
Discovery & handshake GET /api/agent · GET /api/health
List finished runs GET /api/runs
Summary JSON GET /api/runs/{run_id}/summary
Which files landed on disk GET /api/runs/{run_id}/sections
Triage rows (nice for codegen) GET /api/runs/{run_id}/section/triage/table
Single artifact as text GET /api/runs/{run_id}/section/{section}/text
Combined LLM context GET /api/runs/{run_id}/llm.txt
Kick off recon (streaming) POST /api/scan (SSE events)
